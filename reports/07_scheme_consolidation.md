# Scheme Sharing & Consolidation Opportunities

## Principle

Jira Align works best when projects share schemes. Fewer, standardized
schemes = easier Align mapping + less admin overhead.

## Current Scheme Counts

| Scheme Type | Count | Target | Reduction Needed |
|---|---|---|---|
| Workflow Schemes | 82 | 3-5 | 77 |
| Issue Type Schemes | 37 | 3-5 | 32 |
| Issue Type Screen Schemes | 48 | 3-5 | 43 |
| Field Configurations | 26 | 2-3 | 23 |
| Field Config Schemes | 23 | 2-3 | 20 |
| Permission Schemes | 41 | 3-5 | 36 |
| Notification Schemes | 32 | 2-3 | 29 |

## Estimated Effort (by scheme type)

| Scheme Type | Effort Level | Rationale |
|---|---|---|
| Workflows | **CRITICAL / HIGH** | Align status mapping depends on this; requires stakeholder alignment |
| Issue Types | **HIGH** | Must standardize hierarchy for Align; team-managed migration needed |
| Screens | **MEDIUM** | Can consolidate after workflows/issue types stabilize |
| Field Configs | **MEDIUM** | Clean up after custom field audit |
| Permission Schemes | **LOW** | Independent of Align mapping |
| Notification Schemes | **LOW** | Independent of Align mapping |
